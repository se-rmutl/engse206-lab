# Campus Resource Booking — Week 02 Example

> **กรณีศึกษาแกนกลาง:** Campus Resource Booking — ระบบจองพื้นที่ทำงานกลุ่มและอุปกรณ์การเรียนรู้ในมหาวิทยาลัย  
> **วัตถุประสงค์:** แสดงตัวอย่างผลลัพธ์ Week 02 ตามโครง `git-student-template-week02-week03` เพื่อให้นักศึกษาดู “รูปแบบการคิด + รูปแบบไฟล์” ไม่ใช่คัดลอกเนื้อหาไปใช้กับ Case อื่นโดยตรง

## สิ่งที่เรียนรู้จากตัวอย่างนี้

1. เริ่มจากปัญหาและผลกระทบ ไม่เริ่มจากหน้าจอหรือเทคโนโลยี
2. ระบุ stakeholder ให้มากกว่าผู้ใช้หลักเพียงกลุ่มเดียว
3. วาด System Context Diagram เพื่อบอกว่า “ระบบอยู่ตรงไหน” และเชื่อมกับใคร
4. กำหนด scope เพื่อคุมขนาดงาน และเปลี่ยนข้อสงสัยเป็น Open Questions สำหรับ Week 03
5. ใช้ **Draw.io (`.drawio`) เป็น source หลัก** และ export PNG สำหรับแสดงใน Markdown/GitHub

## กติกา Draw.io สำหรับนักศึกษา

- แก้ diagram ที่ไฟล์ `.drawio` ก่อนเสมอ
- ทุกครั้งที่แก้ diagram ให้ Export เป็น PNG แล้วแทนไฟล์ preview เดิม
- อย่าแก้เฉพาะ PNG เพราะจะไม่เหลือ source ที่แก้ไขต่อได้
- ชื่อไฟล์ควรคงตามที่อาจารย์กำหนด เช่น `w02-system-context.drawio` และ `w02-system-context.png`
