# Campus Resource Booking — Week 03 Example

> **กรณีศึกษาแกนกลาง:** Campus Resource Booking — ระบบจองพื้นที่ทำงานกลุ่มและอุปกรณ์การเรียนรู้ในมหาวิทยาลัย  
> **Input จาก Week 02:** Stakeholder Map, System Context, Scope, Assumptions และ Open Questions  
> **Output ของ Week 03:** Elicitation Plan, Interview Guide, Bias Check, Question Rehearsal Notes และ Initial Requirement Candidates

## เป้าหมายของตัวอย่างนี้

ตัวอย่าง Week 03 สาธิตว่าทีมวิศวกรรมซอฟต์แวร์เปลี่ยน “สิ่งที่ยังไม่รู้” ใน Week 02 ให้กลายเป็น “แผนเก็บหลักฐาน” ได้อย่างไร ก่อนเข้าสู่ stakeholder simulation ใน Week 04

> **สำคัญ:** งานใน Week 03 ยังไม่ใช่ SRS และ Initial Requirement Candidates ยังไม่ใช่ requirement ที่ได้รับการอนุมัติแล้ว
