## Rehearsal Summary

### Scenario used

นักศึกษาต้องการจองห้องทำงานกลุ่มเวลา 16:00–18:00 พร้อมโปรเจกเตอร์ แต่ห้องว่างเพียงบางห้องและอุปกรณ์มีคำขอซ้อนอยู่ เจ้าหน้าที่ต้องตรวจว่าเป็นกิจกรรมปกติหรือกิจกรรมที่ต้องอนุมัติพิเศษ

### Questions tested

- Q-01 workflow ปัจจุบัน
- Q-03 approval rule
- Q-04 booking conflict
- Q-05 cancellation/no-show
- Q-06 handover/return
- Q-08 notification pain point

### What was unclear / leading

- Q-03 ต้องเพิ่มคำถามให้ได้ “เงื่อนไข + ตัวอย่างเหตุการณ์จริง” ไม่ใช่คำตอบกว้าง ๆ
- Q-05 ควรแยก late cancellation และ no-show ออกจากกัน
- Q-08 เดิมเสี่ยงทำให้ผู้ตอบเสนอช่องทางแจ้งเตือนแทนการอธิบายปัญหาของช่องทางปัจจุบัน

### Follow-up questions added

1. “ครั้งล่าสุดที่อนุมัติทันทีเกิดขึ้นเมื่อไร และเพราะเหตุใด?”
2. “ถ้าผู้ใช้ไม่มาตามเวลาจอง ทรัพยากรจะถูกนำกลับมาให้คนอื่นใช้เมื่อไร?”
3. “ข้อมูลใดทำให้คุณมั่นใจว่าควรอนุมัติหรือปฏิเสธ?”
4. “ผู้ใช้พลาดสถานะในจุดใดบ่อยที่สุด และผลกระทบคืออะไร?”

### Changes made to the interview guide

- เพิ่ม Q-05a และ Q-05b เพื่อแยก late cancellation/no-show
- เพิ่ม Q-03a เพื่อขอตัวอย่าง approval decision ล่าสุด
- ปรับ Q-08 ให้ถาม workflow การรับข้อมูลก่อนถามเรื่องช่องทาง
- เพิ่ม Q-11 เพื่อทวนความเข้าใจในตอนท้าย

### Evidence rules to use in Week 04

- ให้รหัสหลักฐาน เช่น `E-STAFF-01`, `E-STUDENT-02`, `E-MANAGER-01`
- บันทึก source และสถานการณ์ที่กล่าวถึงทุกครั้ง
- ระบุ type: Fact / Opinion / Preference / Assumption / Proposed Solution
- ไม่แปลงคำพูดเป็น requirement ทันที; ให้นำไปวิเคราะห์เป็น candidate พร้อม traceability
