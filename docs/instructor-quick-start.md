# คู่มือเริ่มต้นสำหรับอาจารย์

## ก่อนเปิดรายวิชา

1. กำหนดจำนวนกลุ่มและสมาชิกต่อกลุ่ม (แนะนำ 3–4 คน)
2. เลือก/สุ่ม Case Cards จาก [Case Project ทั้ง 10 เรื่อง](../cases/README.md)
3. สร้าง repository ต่อกลุ่มจาก `student-repo-starter/` หรือให้กลุ่ม copy starter
4. กำหนดช่องทางส่ง URL, วิธีตั้งชื่อ repo, cutoff และนโยบาย late submission
5. เตรียม role cards / information cards ของแต่ละ case เพื่อใช้ Week 4 simulation
6. แจก infographic และ sample completed work ของ Week 1

## วิธีคุมมาตรฐาน

- จำกัด 3–4 stakeholder roles, 8–12 core functions และ 1–2 main workflows
- ใช้ rubric กลางเดียวกัน
- ตรวจ commit log/Team Worklog ร่วมกับคุณภาพเอกสาร
- ให้ feedback แบบถามย้อน “หลักฐานใดสนับสนุนความต้องการนี้”
- ดึงทีมที่รีบออกแบบหน้าจอกลับมาที่ user task, pain point, scope และ requirement

## เมื่อติดปัญหา

| อาการ | วิธีช่วยที่แนะนำ |
|---|---|
| ขอบเขตใหญ่เกิน | ตัด workflow รองและระบุ Out of Scope |
| Requirement กำกวม | ขอ example, acceptance criteria หรือถาม stakeholder |
| สมาชิกไม่ทำงาน | ตรวจ worklog/commit log และใช้ peer check |
| เสนอเทคโนโลยีก่อนเข้าใจปัญหา | ขอ design rationale ที่อ้างหลักฐาน/constraint |
