# PLO – CLO – Bloom’s Taxonomy Mapping

## PLO ที่รายวิชาขับเคลื่อน

| PLO | Sub-PLO ที่เกี่ยวข้อง | บทบาทในรายวิชา |
|---|---|---|
| PLO1 | 1A, 1B, 1C, 1F | หลักการวิศวกรรมซอฟต์แวร์ จริยธรรม ผลกระทบของซอฟต์แวร์ และการอธิบายความต้องการ |
| PLO2 | 2A, 2C | สืบค้น ตีความ ประเมินสารสนเทศ และวิเคราะห์/ออกแบบองค์ประกอบซอฟต์แวร์ให้ตรงข้อกำหนด |
| PLO3 | 3A, 3B, 3C, 3D, 3F | ประยุกต์ความรู้และเครื่องมือเพื่อแก้ปัญหาซอฟต์แวร์ในบริบทงานจริง |
| PLO4 | 4A | สนับสนุนผ่านการวางแผนชิ้นงาน การจัดการเวอร์ชัน และการตัดสินใจด้านขอบเขต/คุณค่า |
| PLO5 | 5B | สนับสนุนผ่านการสัมภาษณ์ การต่อรอง เอกสาร และการนำเสนอ |

> PLO1–PLO3 เป็นแกนหลักของรายวิชา ส่วน PLO4–PLO5 เป็นผลลัพธ์สนับสนุนจากการทำโครงการกลุ่มและการสื่อสารกับ stakeholder

## CLO ของรายวิชา

| CLO | ผลลัพธ์การเรียนรู้ระดับรายวิชา | Bloom’s ที่เด่น | หลักฐาน |
|---|---|---|---|
| CLO1 | อธิบายและวิเคราะห์บทบาท requirements/design ใน SDLC รวมถึง stakeholder, scope, constraint และ ethics | Understand / Analyze | Quiz, Case Analysis, Problem Brief |
| CLO2 | วางแผนและดำเนินการเก็บ requirement แล้ววิเคราะห์ จัดหมวดหมู่ จัดลำดับ และต่อรองอย่างมีเหตุผล | Apply / Analyze | Elicitation Plan, Interview Guide, Evidence Log, Backlog |
| CLO3 | จัดทำและปรับปรุง SRS, context model, user story/use case, acceptance criteria, NFR และ traceability ที่ชัดเจนและทดสอบได้ | Apply / Create | SRS v1, Models, Traceability Matrix |
| CLO4 | ประเมินคุณภาพ requirement ทวนสอบ จัดการการเปลี่ยนแปลง และติดตามความเชื่อมโยง requirement-to-design | Evaluate | Review Report, Change Request, Impact Analysis |
| CLO5 | ออกแบบระดับเริ่มต้นที่สอดคล้องกับ requirement ครอบคลุม strategy, architecture, component, data, interface และ UX/UI | Create | Design Strategy, Architecture, Prototype, Detailed Design |
| CLO6 | ประเมินคุณภาพ/ความเหมาะสมของการออกแบบและสื่อสาร rationale ต่อทีม/ผู้มีส่วนได้ส่วนเสีย | Evaluate / Create | Design Review, Test Evidence, Pitch, Reflection |
