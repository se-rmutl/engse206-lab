#### **Step 2: Role-Play Interview (30 นาที)**

**วิธีการ:**

**สำหรับ Interviewer (ผู้สัมภาษณ์):**

```
📋 Interview Script:

[เริ่มต้น - 2 นาที]
"สวัสดีครับ/ค่ะ ขอบคุณที่ให้เวลาสัมภาษณ์วันนี้
วันนี้เราอยากเข้าใจการทำงานของคุณมากขึ้น
ไม่มีคำตอบที่ถูกหรือผิด อยากรู้ความคิดเห็นจริงๆ ของคุณ"

[คำถามอุ่นเครื่อง - 3 นาที]
1. บอกเกี่ยวกับตัวเองและงานของคุณหน่อยได้ไหม?
2. ทำงานตำแหน่งนี้มานานเท่าไหร่?
3. วันทำงานปกติของคุณเป็นยังไง?

[คำถามหลัก - 20 นาที]

📍 ถามเกี่ยวกับปัญหาปัจจุบัน:
4. ในการทำงานแต่ละวัน คุณใช้เครื่องมืออะไรบ้าง?
5. ปัญหาที่เจอบ่อยที่สุดในการทำงานคืออะไร?
6. อะไรทำให้คุณรู้สึกหงุดหงิดหรือเครียดที่สุด?
7. สิ่งที่เสียเวลามากที่สุดคืออะไร?

📍 ถามเกี่ยวกับเป้าหมาย:
8. อะไรคือความสำเร็จในการทำงานของคุณ?
9. คุณวัดผลว่าทำงานได้ดียังไงบ้าง?
10. ถ้าจะเปลี่ยนแปลงอะไรได้ 1 อย่าง จะเป็นอะไร?

📍 ถามเกี่ยวกับเทคโนโลยี:
11. คุณคุ้นเคยกับเทคโนโลยีแค่ไหน? (1-5)
12. แอปหรือระบบที่ชอบใช้คืออะไร? ทำไมชอบ?
13. Feature อะไรที่อยากให้มีมากที่สุด?

[ปิดท้าย - 5 นาที]
14. มีอะไรอยากเพิ่มเติมไหม?
15. ถ้าเราพัฒนาระบบใหม่ คุณสนใจทดสอบไหม?

"ขอบคุณมากครับ/ค่ะ!"
```

**💡 Tips สำหรับ Interviewer:**
```
✅ DO:
- ฟังมากกว่าพูด (80% listening)
- ถาม "ทำไม" หลายๆ ครั้งเพื่อเจาะลึก
- ให้เงียบ 3-5 วินาที หลังจาก user พูดจบ (อาจพูดต่อ)
- แสดงความสนใจ (eye contact, nodding)
- บันทึกคำพูดสำคัญ (quotes)

❌ DON'T:
- ถามคำถามแบบ Yes/No
- ใส่ความเชื่อของตัวเอง
- แนะนำ solution ระหว่างสัมภาษณ์
- ขัดจังหวะขณะ user กำลังพูด
- ตัดสินหรือวิจารณ์
```

**สำหรับ Note Taker (ผู้บันทึก):**

```
📝 สิ่งที่ต้องบันทึก:

1. ข้อมูลพื้นฐาน
   - ชื่อบทบาท (Agent, Supervisor, etc.)
   - ประสบการณ์
   - เครื่องมือที่ใช้

2. Pain Points (ปัญหา) - บันทึกให้ละเอียด
   - ปัญหาอะไร?
   - เกิดบ่อยแค่ไหน?
   - Impact ต่องานมากน้อยแค่ไหน?
   
   ตัวอย่าง:
   ❌ "ระบบช้า"  (ไม่เพียงพอ)
   ✅ "ระบบโหลด 30 วินาที ตอนเช้า ทำให้ไม่ทันรับสายตรงเวลา" (ดี)

3. Goals (เป้าหมาย)
   - อยากทำอะไรให้สำเร็จ?
   - วัดความสำเร็จยังไง?
   
   ตัวอย่าง:
   ✅ "อยากเปลี่ยนสถานะได้ภายใน 3 วินาที"
   ✅ "อยากรับสายได้อย่างน้อย 50 สายต่อวัน"

4. Quotes (คำพูดที่น่าสนใจ)
   - บันทึกคำพูดตรงๆ
   - ใช้เครื่องหมาย "..."
   
   ตัวอย่าง:
   💬 "ทุกเช้าฉันต้องรอระบบโหลด ทำให้เครียดว่าจะทันไหม"
   💬 "ถ้าเปลี่ยนสถานะได้เร็วขึ้น ฉันจะทำงานได้มากขึ้น"

5. Tech Behavior
   - ใช้เทคโนโลยีเก่งแค่ไหน? (1-5)
   - ใช้อะไรบ้างในชีวิตประจำวัน?
   - ชอบ/ไม่ชอบเทคโนโลยีอะไร?
```

**สำหรับ User (ผู้ให้สัมภาษณ์):**

```
🎭 วิธีเล่นบทบาท:

1. เข้าถึงตัวละคร
   - นึกถึงคนที่เคยเจอ (แม่, พี่, เพื่อน)
   - อ่าน SRS เพื่อเข้าใจ context
   - คิดถึงปัญหาที่น่าจะเจอ

2. ตอบคำถามแบบสมจริง
   ✅ "ตอนเช้าฉันมักจะกังวลว่าระบบจะเร็วไหม 
       เพราะถ้าช้าจะไม่ทันรับสาย shift แรก"
   
   ❌ "ระบบมันช้า" (สั้นเกินไป)

3. แชร์ความรู้สึก
   - บอกว่ารู้สึกยังไง
   - ทำไมรู้สึกแบบนั้น
   
   ตัวอย่าง:
   💬 "ฉันรู้สึกหงุดหงิดมากเวลาหาปุ่มเปลี่ยนสถานะไม่เจอ
       มันทำให้เสียเวลาและรู้สึกโง่"

4. ใช้ตัวอย่างจริง
   - เล่าเหตุการณ์ที่เคยเจอ
   - ใช้ตัวเลขถ้าเป็นไปได้
   
   ตัวอย่าง:
   💬 "เมื่อวานฉันพลาดข้อความจาก supervisor 
       เพราะ notification ไม่เด่นพอ"
```

#### **Step 3: วิเคราะห์ข้อมูล (15 นาที)**

**สิ่งที่ต้องทำ:**

1. **รวบรวมข้อมูล**
   ```
   - อ่าน notes ทั้งหมด
   - แต่ละคนบอกสิ่งที่สังเกตได้
   - Highlight ข้อความสำคัญ
   ```

2. **จัดกลุ่ม Insights**
   ```
   วิธี: Affinity Mapping
   
   ┌─────────────────────────┐
   │ Theme 1: Time Pressure  │
   ├─────────────────────────┤
   │ • ต้องรีบเปลี่ยนสถานะ       │
   │ • ระบบช้า รอนาน          │
   │ • กังวลเรื่องเวลา          │
   └─────────────────────────┘
   
   ┌─────────────────────────┐
   │ Theme 2: Visibility     │
   ├─────────────────────────┤
   │ • หาปุ่มไม่เจอ             │
   │ • notification ไม่เด่น    │
   │ • สถานะไม่ชัดเจน          │
   └─────────────────────────┘
   ```

3. **สรุป Top 3 Pain Points**
   ```
   เรียงตาม:
   - ความถี่ (เจอบ่อยแค่ไหน)
   - ผลกระทบ (severe แค่ไหน)
   
   ตัวอย่าง:
   1. ระบบโหลดช้า (30 วินาที) - เจอทุกวัน
   2. หาปุ่มเปลี่ยนสถานะไม่เจอ - เจอ 3-4 ครั้ง/วัน
   3. พลาด notification - เจอ 1-2 ครั้ง/วัน
   ```

4. **สรุป Top 3 Goals**
   ```
   เป้าหมายที่สำคัญที่สุด:
   1. เปลี่ยนสถานะได้รวดเร็ว (< 3 วินาที)
   2. อ่านข้อความทันที (real-time)
   3. ดูสถิติตัวเองได้ง่าย
   ```

5. **เลือก Quote ที่ดีที่สุด**
   ```
   เกณฑ์:
   ✓ เป็นตัวแทนความคิดของ user
   ✓ สั้น กระชับ จำง่าย
   ✓ แสดงอารมณ์ความรู้สึก
   
   ตัวอย่าง Quote ที่ดี:
   💬 "ถ้าเปลี่ยนสถานะได้เร็วขึ้น 
       ฉันจะรับสายได้มากขึ้น"
   
   ตัวอย่าง Quote ที่ไม่ดี:
   ❌ "ฉันอยากให้ระบบดีขึ้น" (ทั่วไปเกินไป)
   ```

**💡 Tips การนำเสนอ:**
```

ทุก interface ต้องออกแบบให้ดี

งานนี้อาจดูยากในตอนแรก
แต่ถ้าทำตาม instructions ทีละขั้น
และเริ่มทำเร็ว (ไม่รอวันสุดท้าย)
คุณจะทำได้แน่นอน

Remember:
- เน้น process มากกว่า beauty
- Base on user research (ไม่ใช่ assumption)
- Iterate และ improve
- Ask for help เมื่อต้องการ

Good UX/UI ทำให้ software ของคุณ:
- ใช้งานง่าย
- น่าใช้
- แก้ปัญหาของ users ได้จริง

นี่คือเป้าหมายของเรา! 🎯

Good luck และมีความสุขกับการออกแบบ!
```
