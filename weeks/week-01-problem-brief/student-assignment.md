# งานนักศึกษา Week 1: Problem Brief v0.1

## งานนี้ทำเพื่ออะไร
นี่คือจุดเริ่มต้นของ Semester Project Case ของกลุ่ม เป้าหมายไม่ใช่คิดหน้าจอหรือเลือกภาษาโปรแกรม แต่คือการทำให้ทีมเข้าใจว่า **ปัญหาคืออะไร เกิดกับใคร เหตุใดจึงสำคัญ และระบบเวอร์ชันแรกควรช่วยอะไร**

## รูปแบบงาน
- งานกลุ่ม 3–4 คน
- ใช้ Case Card ที่อาจารย์กำหนด
- ส่งผ่าน Git repository ของกลุ่ม
- ทุกคนต้องบันทึกงานใน Team Worklog และตอบ Exit Ticket รายบุคคล

## ขั้นตอน
1. เปิด Case Card ของกลุ่มจาก [Case Project ทั้ง 10 เรื่อง](../../cases/README.md)
2. สร้าง repository จาก [Student Starter Repository](../../student-repo-starter/README.md)
3. แก้ไฟล์ `docs/01-problem-brief-v0.1.md`
4. ตรวจ [Submission Checklist](submission-checklist.md)
5. commit และ push ก่อน deadline

## เนื้อหาที่ต้องมี
1. ข้อมูลทีมและชื่อโครงการชั่วคราว
2. ภาพรวมปัญหาและบริบท
3. Facts/หลักฐานตั้งต้น และ Pain Points
4. Stakeholder เริ่มต้น
5. เป้าหมายของระบบและ success indicator
6. In Scope / Out of Scope / Constraints / Assumptions
7. ความต้องการผู้ใช้เริ่มต้นอย่างน้อย 5 ข้อ
8. Non-functional concerns อย่างน้อย 3 ข้อ
9. Open questions สำหรับ Week 3–4
10. Team Worklog และการใช้ AI (ถ้ามี)

## สิ่งที่ห้ามทำ
- ห้ามขยายงานเป็นระบบมหาวิทยาลัยทั้งหมด
- ห้ามเขียน requirement จากความคิดของทีมโดยไม่แยก facts/assumptions
- ห้ามเริ่ม design screen หรือ database ก่อนระบุ problem, user, scope และ requirement
- ห้ามใช้ข้อมูลส่วนบุคคลจริง

## iMac M1/macOS และ Windows notebook
ใช้ browser + Google Docs/Microsoft Word/Markdown editor ได้เหมือนกัน แต่ไฟล์หลักต้องเก็บใน Git repository
- **macOS:** Safari/Chrome, Terminal, VS Code หรือ GitHub Desktop
- **Windows:** Edge/Chrome, PowerShell/Git Bash, VS Code หรือ GitHub Desktop
- คำสั่ง Git เหมือนกัน ดู [Git Workflow](../../docs/git-workflow-and-submission.md)
