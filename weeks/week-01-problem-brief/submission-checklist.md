# Submission Checklist — Week 1

## เอกสาร
- [ ] `docs/01-problem-brief-v0.1.md` มีข้อมูลกลุ่มและ Case ID
- [ ] มีภาพรวมปัญหา Facts และ Pain Points
- [ ] มี stakeholder อย่างน้อย 3 บทบาท
- [ ] มี goals และ success indicators
- [ ] แยก In Scope / Out of Scope / Constraints / Assumptions
- [ ] มี user requirements เริ่มต้นอย่างน้อย 5 ข้อ
- [ ] มี NFR concerns อย่างน้อย 3 ข้อ
- [ ] มี open questions สำหรับการเก็บ requirement ต่อ
- [ ] มี Team Worklog และการเปิดเผยการใช้ AI (ถ้ามี)

## Git / การส่งงาน
- [ ] ไฟล์อยู่ใน repository ของกลุ่ม
- [ ] ชื่อ repository ถูกต้องตามรูปแบบที่กำหนด
- [ ] commit message สื่อความหมาย เช่น `docs: complete problem brief v0.1`
- [ ] สมาชิกที่ทำงานหลักมีหลักฐานใน worklog หรือ commit history
- [ ] push ขึ้น remote แล้ว
- [ ] ส่ง URL repository ก่อน deadline

## ถ้าอาจารย์กำหนด
- [ ] PDF export ของ Problem Brief
- [ ] หลักฐานกิจกรรมใน `evidence/week-01/`
- [ ] Exit Ticket รายบุคคล
