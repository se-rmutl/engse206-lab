# Week 09: Midterm Examination

## วัตถุประสงค์
ประเมินผลการเรียนรู้รายบุคคลตาม CLO1–CLO4 โดยใช้ข้อสอบที่วัดการอธิบาย การประยุกต์ การวิเคราะห์ และการประเมิน ไม่ใช่การท่องจำคำจำกัดความ

## ขอบเขต
- ประเมินความเข้าใจและการประยุกต์ใช้ Requirements Engineering ตั้งแต่ problem framing ถึง SRS/traceability
- รูปแบบแนะนำ: ข้อสอบปรนัย/อัตนัยสั้น + วิเคราะห์กรณีศึกษา + จัดลำดับ/วิจารณ์ artefact
- ไม่มี Workshop หรือ LAB ในสัปดาห์นี้

## แนวทางอาจารย์
1. แจ้ง blueprint และตัวอย่างคำสั่งอย่างน้อย 1 สัปดาห์ก่อนสอบ
2. วัดความเข้าใจรายบุคคลโดยไม่ใช้เอกสารกลุ่มเป็นคำตอบแทน
3. ใช้กรณีศึกษาใหม่หรือ Case ที่ดัดแปลง เพื่อวัดการถ่ายโอนความรู้
4. สรุป misconception สำคัญเพื่อนำไป coaching ในสัปดาห์ถัดไปหรือปรับรายวิชารอบหน้า

## สิ่งที่นักศึกษาควรเตรียม
- ทบทวน artefact ของกลุ่มและเหตุผลที่อยู่เบื้องหลัง
- ฝึกอธิบายความต่างของ concept ที่ใกล้กัน
- ฝึกอ่าน requirement/design แล้วหาข้อกำกวม ความขัดแย้ง ความไม่ครบถ้วน หรือ trade-off
